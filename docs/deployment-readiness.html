คู่มือสำหรับเปิดใช้งานจริง version แรก

สว รย รักษ์โลกและสิ่งแวดล้อม

หน้านี้สรุปขั้นตอนเตรียมระบบสะสมแต้มจากการแลกขยะให้อ่านง่ายขึ้น เหมาะสำหรับครูหรือผู้ดูแลที่ต้องตรวจความพร้อมก่อนให้นักเรียนและเจ้าหน้าที่ใช้งานจริง

ก่อนเปิดระบบต้องมีอะไรบ้าง

ใช้รายการนี้ตรวจแบบเร็ว ถ้าครบทุกข้อจึงค่อยเปิดให้ใช้งาน

  • ตั้งค่าไฟล์ .env แล้ว
  • สร้างฐานข้อมูลและบัญชีเริ่มต้นแล้ว
  • ตรวจ /api/health แล้วขึ้นปกติ
  • เปลี่ยนรหัสผ่านบัญชีเริ่มต้นแล้ว
  • วางแผนสำรองข้อมูลเรียบร้อยแล้ว
  • เตรียมข้อมูล
  • ติดตั้ง
  • เปิดระบบ
  • ตรวจสถานะ
  • บัญชีเริ่มต้น
  • ทดสอบก่อนใช้จริง
  • สำรองข้อมูล
1

เตรียมข้อมูลสำคัญ

ขั้นตอนนี้ทำครั้งแรกบนเครื่องหรือ server ที่จะใช้เปิดระบบจริง

ไฟล์ตั้งค่าระบบ

คัดลอกไฟล์ .env.example แล้วเปลี่ยนชื่อเป็น .env จากนั้นตรวจว่ามีค่า DATABASE_URL และ SESSION_SECRET

รหัสลับของระบบ

SESSION_SECRET ห้ามใช้ค่าตัวอย่าง ให้สร้างค่าใหม่ที่ยาวอย่างน้อย 32 ตัวอักษร และเก็บเป็นความลับ 

DATABASE_URL="file:./prod.db"
SESSION_SECRET="replace-with-a-real-secret-before-deploy"

ถ้าต้องสร้างรหัสลับใหม่ ให้ผู้ดูแลที่ถนัด command line รันคำสั่งนี้: 

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
2

ติดตั้งและสร้างฐานข้อมูล

คำสั่งชุดนี้ทำให้ระบบพร้อมมีตารางฐานข้อมูลและข้อมูลเริ่มต้น

  1. 1

    ติดตั้งไฟล์ที่ระบบต้องใช้

    npm ci
  2. 2

    เตรียม Prisma Client

    npm run db:generate
  3. 3

    สร้างหรือปรับฐานข้อมูลให้ตรงกับระบบ

    npm run db:deploy
  4. 4

    เพิ่มบัญชีเริ่มต้นและชนิดขยะพื้นฐาน

    npm run db:seed
ฐานข้อมูลอยู่ตรงไหน

ถ้าใช้ค่า DATABASE_URL="file:./prod.db" ไฟล์ฐานข้อมูลจริงจะอยู่ที่ prisma/prod.db ควรเก็บไว้บนพื้นที่ที่ไม่หายเมื่อ restart server

3

ตรวจ build และเปิดระบบ

ใช้ชุดนี้ก่อนเปิดจริง เพื่อจับปัญหาให้เจอก่อนผู้ใช้งาน

npm run verify
npm run build
npm run start
ข้อควรระวัง

เมื่อนำขึ้นใช้งานจริงควรเปิดผ่าน HTTPS และควรมี reverse proxy หรือ hosting ที่ช่วยจัดการความปลอดภัย ไม่ควรเปิด server ออกอินเทอร์เน็ตโดยตรงแบบไม่มีชั้นป้องกัน

4

ตรวจสถานะระบบ

หลังเปิดระบบ ให้ตรวจว่าเว็บและฐานข้อมูลทำงานได้จริง

http://localhost:3000/api/health
ถ้าปกติ

หน้า health จะตอบกลับประมาณว่า ok: true และ database: ok

ถ้าไม่ปกติ

ถ้าได้ status 503 ให้ตรวจไฟล์ .env, ตำแหน่งไฟล์ฐานข้อมูล และสิทธิ์การเขียนไฟล์ของ server

5

บัญชีเริ่มต้น

ใช้เพื่อเข้าไปเปิดระบบครั้งแรกเท่านั้น

บทบาท ชื่อผู้ใช้ รหัสผ่านเริ่มต้น ต้องทำต่อ
ครูผู้ดูแลระบบ teacher.admin Password123! เปลี่ยนรหัสผ่านทันทีหลัง login ครั้งแรก
เจ้าหน้าที่ทดลอง staff.demo Password123! เปลี่ยนรหัสผ่านหรือปิดบัญชีเมื่อสร้างบัญชีจริงแล้ว
6

เช็กลิสต์ก่อนเปิดให้นักเรียนใช้

ติ๊กทีละข้อหลังทำสำเร็จ เพื่อยืนยันว่าระบบหลักพร้อมครบ

  • เปิด /api/health แล้วสถานะปกติ
  • ครู login เข้าระบบด้วยบัญชีจริงได้
  • เพิ่มหรือ import นักเรียนอย่างน้อย 1 ห้องเรียนได้
  • เจ้าหน้าที่ค้นหานักเรียนด้วยเลขประจำตัวนักเรียนได้
  • เจ้าหน้าที่บันทึกการแลกขยะได้อย่างน้อย 1 รายการ
  • สมาชิกเห็นแต้มสะสมและประวัติของตัวเองได้
  • ครูเห็น dashboard, กราฟ และอันดับแยกห้องเรียนได้
  • ดาวน์โหลด export และ backup จากหน้าครูได้
7

สำรองข้อมูลและความปลอดภัย

ส่วนนี้สำคัญหลังเริ่มมีข้อมูลนักเรียนและประวัติการแลกขยะจริง

ควรทำเป็นประจำ

ดาวน์โหลด backup จากหน้าครู และเก็บสำเนาไว้นอก server หลัก เช่น cloud drive ของโรงเรียนหรือเครื่องสำรอง

ห้ามเปิดเผย

ไฟล์ .env และไฟล์ฐานข้อมูล prod.db มีข้อมูลสำคัญ ห้ามส่งต่อหรือเก็บในที่สาธารณะ

เอกสารนี้ออกแบบให้เปิดอ่านใน browser ได้โดยตรง และสามารถพิมพ์เป็นกระดาษสำหรับตรวจวันเปิดใช้งานจริงได้
